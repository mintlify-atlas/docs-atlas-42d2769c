The app uses **React Navigation v6** (`@react-navigation/native` + `@react-navigation/native-stack`). All navigators are native stack navigators except the tab bar defined in `Tab.jsx`, which uses `@react-navigation/bottom-tabs`.

## Navigator tree

```
NavigationContainer
в””в”Җв”Җ Stack  (Root)
    в””в”Җв”Җ MAIN_STACK  вҶ’  MainStack
        в”ңв”Җв”Җ AUTHED_STACK   вҶ’  AuthStack     (when loggedIn === true)
        в””в”Җв”Җ UNAUTHED_STACK вҶ’  UnAuthStack  (when loggedIn === false)
```

`MainStack` reads `loggedIn` from `useAuthStore` on every render. React Navigation re-evaluates which screen to display whenever that value changes вҖ” no explicit `navigation.navigate` call is needed to switch between the authenticated and unauthenticated flows.

<Note>
The `isFirstLaunch` check (using `AsyncStorage.getItem('alreadyLaunched')`) is already in place. The onboarding screen code is commented out but the infrastructure is ready to re-enable it.
</Note>

## Tab navigator

`Tab.jsx` defines a bottom tab navigator with a fully custom `CustomTabBar`. The tab bar renders inside an `HStack` with a pill-shaped dark container for the three main tabs and a floating map button on the right.

| Tab | Screen | Icon (focused / unfocused) |
|---|---|---|
| Home | `Home` | `TabHomeFocused` / `TabHome` |
| History | `TransactionHistory` | `TabTransactionFocused` / `TabTransaction` |
| Profile | `Profile` | `TabProfileFocused` / `TabProfile` |
| Merchant | `Merchant` | `TabMap` (always, outside pill) |

<Note>
The `TabStack` component is defined in `src/navigation/Tab/Tab.jsx` but is not currently mounted in `Root.jsx`. The `AuthStack` renders `Home`, `Profile`, and `TransactionHistory` as individual stack screens instead.
</Note>

## Triggering navigation

Use the `useNavigation` hook for navigation inside components, and the imperative `navigateRoute` helper for navigation from outside the React tree (e.g., push notification handlers).

```javascript
// Inside a component
import { useNavigation } from '@react-navigation/native';
import { FIND_USER } from '~/utils/enums/routeName';

const { navigate } = useNavigation();
navigate(FIND_USER);

// Passing params
navigate(SUCCESS, { paymentData });
```

```javascript
// Outside the React tree (e.g., notification handler)
import { navigateRoute } from '~/utils/helpers/navigate';

await navigateRoute(REQ_PAYMENT_SUCCESS, { paymentData });
```

The `NavigationContainer` is given a `ref` via `setNavigationRef` from `~/utils/helpers/navigate`, which enables imperative navigation calls.

## Deep links

The `Root` component configures a `linking` object that maps the `DOSS_NOTIFICATIONS` screen to the URL path `Notifications`:

```javascript
const config = {
  screens: {
    [MAIN_STACK]: {
      screens: {
        [AUTHED_STACK]: {
          screens: {
            [DOSS_NOTIFICATIONS]: 'Notifications',
          },
        },
      },
    },
  },
};

const linking = {
  prefixes: [deep_link.pre_fix],
  config,
};
```

## Push notification routing

When a foreground FCM message arrives with `payment_data`, the notification handler in `useForegroundNotifications` navigates directly to the appropriate alert screen without displaying a system notification:

```javascript
switch (status) {
  case 'success':
    return navigateRoute(txn_id ? REQ_PAYMENT_SUCCESS : SUCCESS, { paymentData });
  case 'failed':
    return navigateRoute(txn_id ? REQ_PAYMENT_FAILED : ERROR, { paymentData });
}
```

All other foreground messages are displayed via `notifee.displayNotification`. Tapping a notifee notification opens the deep link URL defined in `deep_link.notification`.

## Screen options

All stacks share the same `screenOptions` object from `~/themes/screenOptions`:

```javascript
export const screenOptions = {
  headerShown: false,
  gestureEnabled: true,
  animation: isIOS ? 'default' : 'none',
};
```

Headers are hidden globally. Individual screens build their own header inside the screen component using `ScreenWrapper` from `~/components/common/ScreenWrapper`.

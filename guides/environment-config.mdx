## API base URL

The app's HTTP client is configured in `src/services/api-config/axios.js`. The base URL is **hardcoded**:

```javascript
export const axios = Axios.create({
  baseURL: 'https://dossapp.com/api/',
  timeout: 30000,
});
```

The previous staging URL (`https://tikkets.com/api`) is commented out in the same file.

### Changing the base URL per environment

React Native does not ship with a built-in `.env` loader. The recommended approach is to use [`react-native-config`](https://github.com/luggit/react-native-config) or a similar package:

<Steps>
  <Step title="Install react-native-config">
    ```bash
    yarn add react-native-config
    cd ios && pod install
    ```
  </Step>

  <Step title="Create environment files">
    ```bash
    # .env.development
    API_BASE_URL=https://staging.dossapp.com/api/

    # .env.production
    API_BASE_URL=https://dossapp.com/api/
    ```
  </Step>

  <Step title="Update axios.js">
    ```javascript
    import Config from 'react-native-config';
    import Axios from 'axios';

    export const axios = Axios.create({
      baseURL: Config.API_BASE_URL,
      timeout: 30000,
    });
    ```
  </Step>
</Steps>

<Note>
  Until environment variables are wired up, switching between staging and production requires manually editing `baseURL` in `src/services/api-config/axios.js`.
</Note>

### Auth token handling

The Axios interceptors in `axios.js` automatically:
- Attach a `Bearer` token from `AsyncStorage` on every request.
- Clear the stored token and reject the promise when the server returns `"Unauthenticated."`.

## Path aliases

The `~` alias resolves to `./src/`. It is configured in two places so both the Babel compiler and TypeScript type checker understand it:

<CodeGroup>

```javascript babel.config.js
module.exports = {
  presets: ['module:@react-native/babel-preset'],
  plugins: [
    [
      'module-resolver',
      {
        root: ['.'],
        alias: {
          '~': './src',
        },
      },
    ],
  ],
};
```

```json tsconfig.json
{
  "extends": "@react-native/typescript-config/tsconfig.json",
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "~/*": ["./src/*"]
    }
  }
}
```

</CodeGroup>

With this setup, any import like `import storage from '~/store/LocalStorage/storage'` resolves to `src/store/LocalStorage/storage`.

## Metro bundler

Metro is configured in `metro.config.js` to support importing SVG files as React components via `react-native-svg-transformer`:

```javascript
const {getDefaultConfig, mergeConfig} = require('@react-native/metro-config');

const defaultConfig = getDefaultConfig(__dirname);
const {assetExts, sourceExts} = defaultConfig.resolver;

const config = {
  transformer: {
    babelTransformerPath: require.resolve('react-native-svg-transformer'),
  },
  resolver: {
    assetExts: assetExts.filter(ext => ext !== 'svg'),
    sourceExts: [...sourceExts, 'svg'],
  },
};

module.exports = mergeConfig(defaultConfig, config);
```

This removes `svg` from `assetExts` (which would treat SVGs as static assets) and adds it to `sourceExts` (which allows transforming them as JavaScript modules).

## ESLint

Linting configuration in `.eslintrc.js`:

```javascript
module.exports = {
  root: true,
  extends: '@react-native',
};
```

The project also has `@tanstack/eslint-plugin-query` (`^5.27.7`) listed as a dev dependency in `package.json`. To enable the TanStack Query lint rules, update `.eslintrc.js`:

```javascript
module.exports = {
  root: true,
  extends: [
    '@react-native',
    'plugin:@tanstack/eslint-plugin-query/recommended',
  ],
  plugins: ['@tanstack/query'],
};
```

Run the linter:

```bash
yarn lint
```

## Prettier

Formatting rules from `.prettierrc.js`:

```javascript
module.exports = {
  arrowParens: 'avoid',
  bracketSameLine: true,
  bracketSpacing: false,
  singleQuote: true,
  trailingComma: 'all',
};
```

| Option | Value | Effect |
|---|---|---|
| `arrowParens` | `'avoid'` | Omit parentheses for single-argument arrow functions: `x => x` |
| `bracketSameLine` | `true` | Place the closing `>` of a JSX element on the last line |
| `bracketSpacing` | `false` | No spaces inside object braces: `{foo}` not `{ foo }` |
| `singleQuote` | `true` | Use single quotes for strings |
| `trailingComma` | `'all'` | Trailing commas everywhere valid in ES5+ |

## TypeScript

TypeScript configuration in `tsconfig.json`:

```json
{
  "extends": "@react-native/typescript-config/tsconfig.json",
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "~/*": ["./src/*"]
    }
  }
}
```

The base configuration is inherited from `@react-native/typescript-config` (`0.73.1`), which enforces strict type-checking settings appropriate for React Native 0.73.

TypeScript version: `5.0.4`

## Jest

Test configuration in `jest.config.js`:

```javascript
module.exports = {
  preset: 'react-native',
};
```

The `react-native` preset configures Jest with the correct Babel transform, module name mappers for assets, and a React Native-compatible test environment.

Run tests:

```bash
yarn test
```

## Native asset linking

`react-native.config.js` registers the custom fonts directory so they are linked to native projects:

```javascript
module.exports = {
  project: {
    ios: {},
    android: {},
  },
  assets: ['./src/assets/fonts'],
};
```

After adding new font files to `src/assets/fonts/`, run:

```bash
npx react-native-asset
```

## Node version

The `engines` field in `package.json` requires:

```json
"engines": {
  "node": ">=18"
}
```

Use [nvm](https://github.com/nvm-sh/nvm) or [fnm](https://github.com/Schniz/fnm) to manage Node versions:

```bash
nvm install 18
nvm use 18
```
